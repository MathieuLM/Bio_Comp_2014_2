Item (a): numero de subsequencias de tamanho 5

O objetivo é de calcular a quantidade de differentes subsequencias de tamanho 5 num arquivo fornecido.

Assim, vamos usar o algoritmo seguinte, e implementar em Python dois versoes:

-a primeira usa a estrutura de dados List;

-a segunda usa um dicionario.

Algoritmo em pseudo-codigo:

  1. Read a sequence of length 5;
    • if first encounter, add it to the collection;
  2. Repeat step 1 until the end of the file

No caso de List, usarmos o metodo count() para detectar a presença na coleçao, e o metodo append() para adicionar na lista.

No caso de Dict, usarmos o metodo "in" e "=".

O resultado é de 1.024 subsequencias (o mesmo pelas dois versoes). A diferença é que a versao de List demorou alguns minutos, e a versao de Dict alguns segundos.

Item (b): numero de subsequencias separadas por "TTT"

O objetivo é de usar a regiao de corte "TTT" como um separador de subsequencias, e assim calcular a quantidade de subsequencias no arquivo.

Algoritmo em pseudo-codigo:

  1. Read a character;
    • if a 'T', increment counter_T;
      • if counter_T is 3, increment counter_seq & reset counter_T;
    • if not a 'T', reset counter_T;
  2. Repeat step 1 until no more characters

O résultado final é de 1.443.795 subsequencias entre separadores "TTT".

Funcionalidades

As funcionalidades implementadas funcionam bem, e são suficientes para usar o sistema numa situação real.

Contudo, umas outras funcionalidades poderiam ser implementadas para ter um melhor aplicativo, como:

- poder adicionar comentários a um ponto

- especificar para ir em qual direcção é o ponto

- poder escolher a confiabilidade dos usuários

- dar uma importância maior ao voto dum usuário confiável

- fazer estatísticas sobre as áreas mais eficientes

- decrementar a confiabilidade dum ponto se a ultima recomendação é antiga demais

Desempenho

O desempenho parece bom, mas não fizemos de testes com um grande número de clientes.

Uma crítica que podemos fazer é que o servidor não identifique individualmente cada cliente, assim ele não sabe quem mandou a informação dum novo ponto, e notifica ele de volta, ainda que não serve para nada.
